2. EXTENSION TABLE EXISTANTE — SPRINT 5 (Suivi GPS)

La table shipments reçoit des colonnes (Sprint 5 — cartographie).
À représenter sur le diagramme final par une note ou un compartiment
« Suivi GPS » sur l'entité shipments :

| Attribut              | Type SQL   | Description                          |
|-----------------------|------------|--------------------------------------|
| origin_lat            | FLOAT      | Latitude origine                     |
| origin_lng            | FLOAT      | Longitude origine                    |

┌─────────────────────────────────────────────────────────────────────────────┐
│ TABLE : assistant_sessions             │ Session chatbot                │
└─────────────────────────────────────────────────────────────────────────────┘

| Attribut          | Type SQL     | Contraintes                            |

6. NOUVEAUX LIENS (Release 3)
--------------------------------------------------------------------------------

  (L) users ──< assistant_sessions
      • assistant_sessions.user_id → users.id | CASCADE
      • 1 utilisateur → 0..* sessions assistant

  (M) assistant_sessions ──< assistant_messages
      • assistant_messages.session_id → assistant_sessions.id | CASCADE
      • 1 session → 0..* messages (user / assistant)

  (N) users ──< assistant_avatars  (rôle admin)
      • assistant_avatars.admin_id → users.id | CASCADE
      • 1 administrateur configure 0..* avatars (un seul is_active à la fois en métier)

TABLEAU 3.8 – Description textuelle de cas d'utilisation « Suivre ses expéditions »
Sprint 2 — GlobalTradeX
================================================================================

+------------------+------------------------------------------------------------------+
| Nom du CU        | « Suivre ses expéditions »                                       |
+------------------+------------------------------------------------------------------+
| Acteurs          | Importateur, Exportateur                                         |
+------------------+------------------------------------------------------------------+
| Résumé           | L'acteur consulte la liste de ses dossiers d'expédition et       |
|                  | accède au détail pour connaître le statut, la route et         |
|                  | l'avancement du traitement.                                      |
+------------------+------------------------------------------------------------------+
| Pré-conditions   | L'acteur est authentifié.                                         |
|                  | Au moins une expédition existe pour cet acteur (owner ou           |
|                  | exportateur rattaché).                                           |
|                  | L'acteur ouvre « Expéditions » (/dashboard/shipments) depuis      |
|                  | la barre latérale.                                               |
+------------------+------------------------------------------------------------------+

Scénario nominal
----------------
 1. Le système appelle GET /api/shipments (filtre mine_only ou exportateur selon rôle).
 2. Le système affiche un tableau : référence GTX, origine, destination, statut,
    dates, indicateur de progression.
 3. L'acteur parcourt la liste et sélectionne une expédition.
 4. Le système charge GET /api/shipments/{id} et affiche la fiche détail.
 5. La fiche présente : informations de route, fret, statut actuel, importateur /
    exportateur associés, barre de progression, liens vers suivi GPS (si initialisé).
 6. L'acteur peut revenir à la liste ou accéder au cockpit métier pour une vue synthèse.

Enchaînements alternatifs
-------------------------
 1.1  Aucune expédition pour l'acteur :
      le système affiche un état vide invitant à créer une première expédition
      (lien vers /dashboard/shipments/new).

 4.1  Expédition introuvable ou accès non autorisé :
      le système affiche une erreur HTTP 404 ou 403 et redirige vers la liste.

 5.1  Statut en « customs_hold » ou autre état bloquant :
      la fiche affiche le statut en évidence ; les actions de changement de statut
      restent réservées au transitaire (Sprint 3).

Post-conditions
---------------
 L'acteur dispose d'une visibilité à jour sur ses expéditions.
 Il peut préparer les étapes suivantes (documents, collaboration) dans les sprints 3 et 4.

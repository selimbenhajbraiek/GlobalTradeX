================================================================================
2.3 Technologies et outils de développement
Projet : GlobalTradeX — Plateforme de pilotage du commerce international
================================================================================

Les frameworks, bibliothèques et services cloud retenus ont un impact direct sur la
productivité de l'équipe : ils accélèrent le développement, structurent le code et
améliorent sa maintenabilité. Cette section présente les produits, outils et langages
effectivement employés dans le dépôt logiciel globaltradeX (API Python, interface
Next.js, base MySQL).


--------------------------------------------------------------------------------
2.3.1 Développement Back-end
--------------------------------------------------------------------------------

FastAPI

FastAPI est un framework web Python moderne, performant et orienté vers la construction
d'API REST. Il s'appuie sur les annotations de type Python et Pydantic pour valider
automatiquement les entrées et sorties JSON. Dans GlobalTradeX, FastAPI expose l'ensemble
des routes métier sous le préfixe /api : authentification (/api/auth), expéditions
(/api/shipments), produits, documents, messagerie, notifications, analytics et assistant.
Le serveur ASGI est démarré
avec Uvicorn pour prendre en charge les requêtes concurrentes en environnement de
développement et de déploiement.


SQLAlchemy 2 & MySQL

SQLAlchemy est un ORM (Object-Relational Mapping) Python qui mappe les entités métier
vers des tables relationnelles : User, Shipment, Product, Document, MessageThread,
Notification, etc. La version 2.x offre une API déclarative claire et des requêtes
typées. La base de données retenue est MySQL, hébergée localement via XAMPP en phase
de développement et de démonstration (soutenance). La connexion s'effectue par le
connecteur PyMySQL. Les évolutions de schéma (nouvelles colonnes, tables de messagerie,
préférences de notification, vérification e-mail) sont versionnées avec Alembic afin
de garantir la reproductibilité des migrations entre environnements.


Sécurité, authentification et e-mails

L'authentification repose sur des jetons JWT (bibliothèque python-jose) délivrés après
connexion ; les mots de passe sont hachés avec bcrypt via passlib. Des tokens à usage
unique sécurisent la vérification d'adresse e-mail et la réinitialisation de mot de
passe. Les e-mails transactionnels (confirmation d'inscription, reset password) sont
envoyés par SMTP via le service MailerSend. Le contrôle d'accès par rôle (importateur,
exportateur, transitaire, courtier, administrateur) est appliqué dans les dépendances
FastAPI (require_role) et miroir côté frontend (RoleGuard, middleware Next.js).


Intelligence artificielle et services externes

L'API Google Gemini 2.5 Flash (client compatible OpenAI) est intégrée pour l'analyse
automatique des documents douaniers, l'assistant conversationnel TradeFlow, les
suggestions d'itinéraire et la synthèse exécutive de la BI prédictive.
En option, l'API HeyGen permet la génération d'un avatar vidéo pour l'assistant
administrable depuis le cockpit admin. Les fichiers uploadés (PDF, images) sont stockés
sur le système de fichiers du serveur (dossier uploads/) avec une taille maximale de
10 Mo par pièce.


--------------------------------------------------------------------------------
2.3.2 Développement Front-end
--------------------------------------------------------------------------------

Next.js 14 & React 18

Next.js est un framework React développé par Vercel. La version 14, avec l'App Router,
structure l'application GlobalTradeX en pages et layouts : landing publique, flux
d'authentification (inscription, vérification e-mail, reset password) et tableau de
bord multi-rôles (/dashboard/importateur, exportateur, transitaire, courtier, admin).
React 18 fournit le modèle de composants réutilisables (cockpits métier, formulaires
multi-étapes de création d'expédition, tableaux, panneaux de détail). Le rendu côté
client (« use client ») est privilégié pour les écrans interactifs consommant l'API.


Tailwind CSS & design éditorial

Tailwind CSS est un framework utility-first qui accélère la mise en forme responsive
sans feuilles de style monolithiques. GlobalTradeX adopte un design system éditorial
(couleurs brass/mist, typographie display, cartes KPI, barre latérale) cohérent sur
tous les cockpits. Les icônes Lucide React complètent l'interface. PostCSS et
Autoprefixer assurent la compatibilité navigateur.


Communication API & sécurité client

Axios est la bibliothèque HTTP utilisée pour appeler l'API FastAPI. Des intercepteurs
attachent le jeton JWT (stocké en cookie) aux requêtes authentifiées et gèrent les
erreurs 401/403. Le middleware Next.js redirige les utilisateurs non connectés vers
/login et oriente chaque rôle vers son cockpit après authentification.


Cartographie et visualisation

Leaflet et react-leaflet affichent le suivi GPS simulé des expéditions sur une carte
interactive. Chart.js et react-chartjs-2 alimentent les graphiques des cockpits
(analytics importateur, exportateur, admin). Des cartes SVG (Natural Earth, topojson)
illustrent le réseau mondial des flux dans la vue administrateur (ShipmentNetworkMap).


Internationalisation

Un LocaleContext et des fichiers de traduction (messages/translations.js) prévoient
plusieurs langues (français, anglais, arabe) pour les libellés de navigation et les
textes communs, en préparation d'une ouverture internationale de la plateforme.


--------------------------------------------------------------------------------
2.3.4 Stack Business Intelligence (spécialité BI)
--------------------------------------------------------------------------------

GlobalTradeX intègre la BI dans l'application (pas d'outil BI séparé type Power BI).

Moteur analytique (back-end)
  - analytics.py : agrégations SQLAlchemy, KPIs réseau, endpoints /api/analytics/*
  - bi_delay_prediction.py : BI prédictive (score retard, corridors, transitaires)
  - Gemini 2.5 Flash : synthèse exécutive en langage naturel (optionnelle)

Visualisation (front-end)
  - Chart.js + react-chartjs-2 : graphiques des cockpits et page analytics
  - Leaflet : cartographie GPS et flux
  - PredictiveBiPanel.jsx, TradeAnalyticsPage.jsx : restitution BI prédictive

Non retenu (justification) : Power BI, Apache Superset, Metabase — déploiement et
licence supplémentaires, dashboards découplés du workflow multi-rôles ; entrepôt DW
physique reporté (ELT on-demand suffisant en PFE). Détail : 06_STACK_BI_ET_JUSTIFICATION.md
et TABLEAU 2.4 (technologies/TABLEAU_2_4_Stack_BI_GlobalTradeX.txt).

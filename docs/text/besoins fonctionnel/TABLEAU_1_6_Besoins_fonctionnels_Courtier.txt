================================================================================
TABLEAU 1.6 – Besoins fonctionnels de l'acteur « Courtier en douane »
Projet : GlobalTradeX — Plateforme de pilotage du commerce international
================================================================================

+----------------------------------+----------------------------------------------------------+
| Fonctionnalité                   | Description                                              |
+----------------------------------+----------------------------------------------------------+
| S'authentifier                   | Pour accéder à son espace, le courtier s'authentifie     |
|                                  | avec son adresse e-mail et son mot de passe. L'accès     |
|                                  | n'est accordé qu'après vérification de l'adresse e-mail. |
|                                  | Un jeton JWT sécurisé ouvre le cockpit douane            |
|                                  | (/dashboard/courtier).                                   |
+----------------------------------+----------------------------------------------------------+
| Récupérer mot de passe           | En cas d'oubli, le courtier demande un lien de           |
|                                  | réinitialisation par e-mail (SMTP). Il définit alors un  |
|                                  | nouveau mot de passe via une page dédiée sécurisée.      |
+----------------------------------+----------------------------------------------------------+
| Gérer profil                     | Le courtier peut consulter et mettre à jour ses          |
|                                  | informations personnelles (nom complet, téléphone,       |
|                                  | avatar) depuis l'espace Profil.                        |
+----------------------------------+----------------------------------------------------------+
| Consulter cockpit douane         | Permet d'afficher un tableau de bord métier : indicateurs|
|                                  | (déclarations en attente, validées aujourd'hui, alertes  |
|                                  | conformité), filtres par juridiction et par statut de    |
|                                  | dossier, et synthèse des anomalies détectées.            |
+----------------------------------+----------------------------------------------------------+
| Consulter file de revue          | Permet d'accéder à la liste des documents non encore     |
|                                  | validés (/documents/pending-review), triés par date de   |
|                                  | dépôt, avec référence d'expédition et auteur du fichier. |
+----------------------------------+----------------------------------------------------------+
| Consulter expéditions            | Permet de parcourir l'ensemble des dossiers d'expédition |
|                                  | afin de corréler les pièces douanières au contexte       |
|                                  | (route, statut customs_hold, parties prenantes).         |
+----------------------------------+----------------------------------------------------------+
| Consulter documents              | Permet de visualiser tous les documents du réseau        |
|                                  | (facture, packing list, certificat d'origine, B/L, etc.)   |
|                                  | depuis l'espace Documents, avec résultat d'analyse IA.   |
+----------------------------------+----------------------------------------------------------+
| Vérifier document (IA)           | Permet de lancer une analyse automatique (OpenAI) sur    |
|                                  | un fichier déposé : détection d'anomalies, champs        |
|                                  | manquants, cohérence douanière. Le résultat est stocké   |
|                                  | dans ai_result et affiché avant décision humaine.        |
+----------------------------------+----------------------------------------------------------+
| Valider document                 | Permet d'approuver une pièce après revue (humaine et/ou  |
|                                  | IA) : le drapeau is_verified passe à vrai, la date de    |
|                                  | validation est enregistrée et l'auteur du fichier est    |
|                                  | notifié.                                                 |
+----------------------------------+----------------------------------------------------------+
| Rejeter document                 | Permet de refuser une pièce non conforme en enregistrant |
|                                  | un motif de rejet (rejection_reason) dans ai_result ;    |
|                                  | l'exportateur ou le déposant est alerté pour correction.   |
+----------------------------------+----------------------------------------------------------+
| Consulter validations du jour    | Permet de lister les documents validés dans la journée   |
|                                  | (endpoint recently-verified-today) pour le reporting     |
|                                  | d'activité du courtier.                                  |
+----------------------------------+----------------------------------------------------------+
| Télécharger documents            | Permet de récupérer le fichier source d'une pièce pour   |
|                                  | archivage ou contrôle approfondi hors ligne.             |
+----------------------------------+----------------------------------------------------------+
| Filtrer par juridiction / statut | Depuis le cockpit, permet de restreindre la vue aux      |
|                                  | dossiers d'une zone douanière ou à un état de déclaration |
|                                  | (en attente, validé, rejeté, alerte).                    |
+----------------------------------+----------------------------------------------------------+
| Suivi GPS en direct              | Permet de localiser sur une carte les expéditions en       |
|                                  | cours, notamment celles en customs_hold nécessitant une  |
|                                  | intervention douanière.                                  |
+----------------------------------+----------------------------------------------------------+
| Consulter liste notifications    | Permet de consulter les alertes liées aux dossiers en    |
|                                  | revue et aux décisions rendues.                          |
+----------------------------------+----------------------------------------------------------+
| Gérer préférences notifications  | Permet de configurer les canaux et types d'alertes       |
|                                  | depuis les paramètres du compte.                         |
+----------------------------------+----------------------------------------------------------+
| Messagerie collaborative         | Permet d'échanger avec l'importateur, l'exportateur ou   |
|                                  | le transitaire pour demander des pièces complémentaires  |
|                                  | ou confirmer une mainlevée.                              |
+----------------------------------+----------------------------------------------------------+
| Procurer aide (assistant IA)     | Un assistant conversationnel (TradeFlow) aide le courtier|
|                                  | sur les réglementations, les types de documents requis   |
|                                  | et l'interprétation des résultats d'analyse.             |
+----------------------------------+----------------------------------------------------------+
| Se déconnecter                   | Permet de clôturer la session en supprimant le jeton     |
|                                  | d'authentification depuis le profil ou la barre latérale.|
+----------------------------------+----------------------------------------------------------+

Remarques :
- Seul le courtier (et l'administrateur) peut valider ou rejeter officiellement un document et
  déclencher l'analyse IA documentaire dédiée à la douane.
- Le courtier ne modifie pas le statut logistique des expéditions (réservé au transitaire).
- Scénario de démonstration : Amira Ben Brahim — GTX-20240315-00042, certificat d'origine
  signalé par l'IA (tampon chambre manquant) puis mainlevée après correction.
